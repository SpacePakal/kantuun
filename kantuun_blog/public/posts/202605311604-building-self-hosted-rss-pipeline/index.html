RSS → SQLite → Claude → Obsidian

So you’re tired of drowning in RSS noise — CVE roundups, vendor fluff, “top 10 tips” garbage — and just want the good stuff: deep offensive research, EDR internals, novel malware analysis. This post walks through a pipeline I built that ingests RSS feeds, scores and summarizes items using the Claude API, and drops polished notes straight into an Obsidian vault.

The full stack:

  • feedparser for RSS ingestion
  • SQLite as a local source-of-truth and dedup layer
  • Claude API for two-stage filtering (cheap scoring → full summarization)
  • Obsidian as the final knowledge base

Architecture Overview

RSS Feeds
    │
    ▼
feedparser (ingest)
    │
    ▼
SQLite (dedup + state)
    │
    ▼
Claude claude-haiku-4-5 (score 1–10)
    │  filter: score >= 7
    ▼
Claude claude-sonnet-4-6 (summarize)
    │
    ▼
Obsidian Vault (.md files)

The two-stage Claude approach keeps costs low — Haiku handles the volume, Sonnet only touches items that pass the threshold.

Step 1: Feed Ingestion with feedparser

First, install dependencies:

pip install feedparser anthropic sqlite-utils

Define your curated feed list. Quality over quantity here — these are sources that consistently publish deep technical material:

FEEDS = [
    "https://posts.specterops.io/feed",
    "https://googleprojectzero.blogspot.com/feeds/posts/default",
    "https://research.nccgroup.com/feed/",
    "https://www.elastic.co/security-labs/rss/feed.xml",
    "https://blog.trailofbits.com/feed/",
]

Ingestion loop:

import feedparser
import hashlib
from datetime import datetime

def ingest_feeds(feeds: list[str]) -> list[dict]:
    items = []
    for url in feeds:
        feed = feedparser.parse(url)
        for entry in feed.entries:
            item = {
                "id": hashlib.sha256(entry.link.encode()).hexdigest()[:16],
                "title": entry.get("title", ""),
                "url": entry.get("link", ""),
                "summary": entry.get("summary", ""),
                "published": entry.get("published", datetime.utcnow().isoformat()),
                "source": feed.feed.get("title", url),
            }
            items.append(item)
    return items

Step 2: SQLite for Deduplication and State

Use sqlite-utils to keep things simple. The schema tracks ingestion state and scores so you can re-run without reprocessing everything.

import sqlite_utils

DB_PATH = "intel.db"

def init_db() -> sqlite_utils.Database:
    db = sqlite_utils.Database(DB_PATH)
    if "items" not in db.table_names():
        db["items"].create({
            "id": str,
            "title": str,
            "url": str,
            "summary": str,
            "published": str,
            "source": str,
            "score": int,
            "processed": int,  # 0 = new, 1 = scored, 2 = summarized
        }, pk="id")
    return db

def upsert_items(db: sqlite_utils.Database, items: list[dict]):
    for item in items:
        if not db["items"].get(item["id"], default=None):
            item["score"] = 0
            item["processed"] = 0
            db["items"].insert(item)

Step 3: Two-Stage Claude Filtering

Stage 1 — Haiku Scoring

Score each item 1–10 based on technical depth. The prompt is deliberately narrow — we’re optimizing for offensive research and internals, not news.

import anthropic

client = anthropic.Anthropic()

SCORE_SYSTEM = """You are a cybersecurity research filter. Score articles 1-10 based on:
- 8-10: Deep technical content — offensive tooling, EDR internals, novel malware analysis, exploit research, red team tradecraft
- 5-7: Useful but general — incident reports, defensive architecture, solid CTF writeups
- 1-4: Noise — CVE roundups, vendor marketing, opinion pieces, "top 10" lists

Respond with ONLY a single integer. No explanation."""

def score_item(item: dict) -> int:
    prompt = f"Title: {item['title']}\n\nSummary: {item['summary'][:500]}"
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=5,
        system=SCORE_SYSTEM,
        messages=[{"role": "user", "content": prompt}],
    )
    try:
        return int(response.content[0].text.strip())
    except ValueError:
        return 0

Stage 2 — Sonnet Summarization

Only items scoring >= 7 make it here. Sonnet produces a structured summary formatted for Obsidian.

SUMMARY_SYSTEM = """You are a cybersecurity analyst creating research notes for an Obsidian vault.
Produce a structured markdown summary with these sections:
- **TL;DR** (2-3 sentences)
- **Key Techniques** (bullet list)
- **Why It Matters** (1-2 sentences, red team / detection angle)
- **Tags** (comma-separated, e.g. #edr-evasion #malware #windows)

Be terse. No fluff. Prioritize actionable technical detail."""

def summarize_item(item: dict) -> str:
    prompt = f"Title: {item['title']}\nURL: {item['url']}\n\nContent:\n{item['summary'][:2000]}"
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=600,
        system=SUMMARY_SYSTEM,
        messages=[{"role": "user", "content": prompt}],
    )
    return response.content[0].text.strip()

Step 4: Writing to Obsidian

Each summarized item becomes a .md file in your vault under Intelligence/. The frontmatter makes it queryable via Dataview.

import os
import re
from pathlib import Path

VAULT_PATH = Path.home() / "Documents" / "Codex" / "Intelligence"

def slugify(title: str) -> str:
    return re.sub(r"[^a-z0-9-]", "-", title.lower())[:60].strip("-")

def write_note(item: dict, summary: str):
    VAULT_PATH.mkdir(parents=True, exist_ok=True)
    filename = f"{item['published'][:10]}-{slugify(item['title'])}.md"
    filepath = VAULT_PATH / filename

    frontmatter = f"""---
title: "{item['title']}"
source: "{item['source']}"
url: "{item['url']}"
published: {item['published'][:10]}
score: {item['score']}
tags: [intel, unreviewed]
---

"""
    filepath.write_text(frontmatter + summary)
    print(f"  ✓ {filename}")

Step 5: Wiring It All Together

def run_pipeline():
    db = init_db()

    print("[1/4] Ingesting feeds...")
    items = ingest_feeds(FEEDS)
    upsert_items(db, items)
    print(f"      {len(items)} items ingested")

    print("[2/4] Scoring with Haiku...")
    unscored = list(db["items"].rows_where("processed = 0"))
    for item in unscored:
        score = score_item(item)
        db["items"].update(item["id"], {"score": score, "processed": 1})
        print(f"      [{score}/10] {item['title'][:60]}")

    print("[3/4] Summarizing with Sonnet...")
    high_value = list(db["items"].rows_where("processed = 1 AND score >= 7"))
    print(f"      {len(high_value)} items passed threshold")
    for item in high_value:
        summary = summarize_item(item)
        write_note(item, summary)
        db["items"].update(item["id"], {"processed": 2})

    print("[4/4] Done.")

if __name__ == "__main__":
    run_pipeline()

Automating with a Cron Job

Run this daily at 8 AM:

crontab -e

0 8 * * * cd /opt/intel-pipeline && /usr/bin/python3 pipeline.py >> /var/log/intel-pipeline.log 2>&1

Or as a systemd timer if you prefer:

# /etc/systemd/system/intel-pipeline.timer
[Unit]
Description=Daily cybersecurity intel pipeline

[Timer]
OnCalendar=*-*-* 08:00:00
Persistent=true

[Install]
WantedBy=timers.target

Querying in Obsidian with Dataview

Once notes are landing in your vault, use a Dataview block to surface recent high-value items:

```dataview
TABLE source, score, published
FROM "Intelligence"
WHERE score >= 8
SORT published DESC
LIMIT 20
```

What’s Next

A few directions worth exploring from here:

  • Vector embeddings — store embeddings in SQLite via sqlite-vec and do semantic dedup across items
  • Hypothes.is integration — pull in your web annotations as additional intel sources
  • GPU-local scoring — replace Haiku with a local model via Ollama for the scoring stage (Mistral 7B works well), keeping Sonnet only for summarization
  • Slack/Discord webhook — push the day’s top-scored items to a channel for async review

The SQLite-as-source-of-truth pattern pays dividends over time — you can backfill scores, re-summarize with better prompts, or add new fields without re-ingesting everything.
